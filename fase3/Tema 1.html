Etiquetas semanticas


"es el uso de las etiquetas HTML para reforzar la semántica, o el significado, de la información en las páginas web más que simplemente redefinir su forma de presentación (apariencia). El HTML semántico es procesado por los navegadores web regulares así como por muchos otros agentes de usuarios. Las CSS son usadas para sugerir la presentación de la página a usuarios humanos" (HTML semanticas, 2019,Wikipedia, La enciclopedia)


Estos elementos,denominados generalmente elementos semánticos sirven para dividir un documento en partes lógicas, indicando el tipo de contenido que se incluye en cada parte, representan por tanto secciones lógicas o componentes de una aplicación web o un documento" (Diez, T., Domínguez, M. J., Martínez, J. J., & Sáenz, J. (2012). Creación de páginas Web accesibles con HTML5)


"Aunque no son las unicas estas son las etiquetas de infraestructura mas importantes:

  • header: representa la cabeera de un documento o seccion; deberian usarse para informacion introductora. Puede haber mas de uno en el documento.
  • footer: representa el cierre de un documento o seccion. Suele contener informacion sobre el autor, copyright, terminos de uso, informacion de contacto, etc
  • nav: Representa la seccion que no tiene los enlaces de navegacion por el estil web.
  • section: define una seccion del documento
  • article: representa un contenido independiente y acontenido, que se podria distribuir de manera independiente del resto del documento (posts de blog, noticias, comentarios, etc.).
  • aside: define contenido que se encuentra fura del lugar en el que esta contenido (cmo barra lateral), por lo que suele estar relacionado con el elemento que lo contiene.
  • figure & figcaption: define un elemento independiente, como una imagen, un fragmento de codigo, un diagrama, etc. cuyo contenido esta relacionado con el elemento que lo contiene, pero cuya posicion es independiente de mismo. (figcaption) es el titulo o la leyenda de este elemento
  • details: Representa una serie de detalles adicionales que el usuario puede ocultar".
